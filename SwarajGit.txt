0) check status-
git status

1) Initialize one repo-
git init

2) Add files-
git add xyz.html [too add a single file]
or
git add -A [too add all the files]

3) Commit files-
git commit -m "This is the commit message"
or
git commit [opens vim editor, then i for inserting, then press escape and ":wq" and enter]

4) change back to the previously saved commit if modified-
git checkout abc.html
or
git checkout -f [make files as last commit]

5) Check Logs-
git log [all commits]
or
git log -p -1 [last one commit and q to quit]

6) Difference between staging and working tree-
git diff
or
Difference between staging and last commit-
git diff --staged

7)Commit skipping staging area(add)-
git commit -a -m "comment here"

8) Remove file if required-
git rm --cached abc.html[remove from only staging area, don't delete]
or
git rm abc.html

9) Summarized status-
git status -s

10) Git Inore-
create a .gitignore file
add the file name eg. [abc.logs] inside the gitignore file
or
add the file name with slash eg.[/abc.logs] inside the gitignore file
if we want to ignore only file with that name inside the same folder 
where the gitignore file is present
or
add [*.log] to ignore all files with that extension
or
add [foldername/] to ignore a directory

11) Add branch-
git branch [gives branch status, where you are present]
or
git branch feature1 [create a branch named feature1]

12) Switch branch-
git checkout feature1 [switch to branch feature1]
